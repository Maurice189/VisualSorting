Abstract

Shell sort is an in-place but not stable comparison sort. It can be seen as either a generalization of bubble sort or insertion sort. The method starts by sorting elements far apart from each other and progressively reducing the gap between them. Starting with far apart elements can move some out-of-place elements into position faster than a simple nearest neighbor exchange. Shell sort is like bubble sort an adaptive sorting algorithm, which means it terminates faster when the list is nearly sorted. In real world applications shell sort performs slower than quick sort, because it has higher cache miss ratios. However due to its little code size it can be considered in embedded system applications.
