Abstract

Selection sort is an in-place algorithm with O(n2) time complexity. In each iteration the smallest element of the unsorted list is calculated and afterwards added to the already sorted list. Conceptually this makes selection sort very similar to insertion sort. However selection sort has to scan through all elements to find the minimum, whereas insertion sort only scans as many elements as it needs in order to place to element to the correct position. Despite the unfavorable efficiency aspects, selection sort performs only n write operations in total, which can be crucial fact if write operations are rather expensive (EEPROM or Flash).
